## 本日学んだこと / 振り返り
Javaには文字列を特定したり、検索したり、文字列同士を連結したりするメソッドが多く用意されている様で、例えば`equals`メソッドなどは下記のように文字列同士が等価なのか判定することができます。  
```
Hero h1 = new Hero("ミナト");
Hero h2 = new Hero("ミナト");

h1.equals(h2); //true
```
もしく下記の様に`==`でも文字列同士の判定ができる様なのですが、この場合は変数同士のアドレスが同一かを判定している様です。  
```
Hero h1 = new Hero("ミナト");
Hero h2 = h1;

h1 == h2 //true
```
Javaには仮想的なメモリ領域という考えがあるようで、`new`などを使ってインスタンス化などをするとメモリ領域の特定のアドレスを占領して、その情報を埋め込む様です。
上記の場合を分解して考えると次の様な形になります。  
```
Hero h1 = new Hero("ミナト");  //メモリ領域2132番地に、Heroインスタンスが代入されたHero型のh1という変数の情報を埋め込む。
Hero h2 = h1;  //Hero型のh2という変数に、メモリ領域2132番地のh1という変数を参照権を与える（エイリアスの様な感じ？）。

2132番地(h1) == 2132番地(h2) //true
```
